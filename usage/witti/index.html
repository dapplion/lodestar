    Connect to Witti testnet¶

    Running a Lodestar node on the Witti multi-client testnet only requires basic familiarity with the terminal.

    Make sure lodestar in installed in your local environment. The following command should return a non error message. If it fails, go the install guide and make sure Lodestar is available in your environment.

    yarn run cli --help

    Eth1 genesis state processing is currently slow, its best to start from known genesis state and disable eth1 processing. First, download a genesis file locally with the command below or from this link.

    wget https://github.com/goerli/witti/raw/master/lighthouse/genesis.ssz

    Make sure that the downloaded genesis file is at the expected path and matches the path in the command below.

    We recommend initializing lodecli into a specific directory, which is helpful for running multiple nodes. Declare a root dir so all data will be installed/fetched by either:

    • set root-dir cli flag or
    • set LODECLI_ROOT_DIR env variable

    Info

    All paths are relative to the working directory when running lodecli

    To initialize the Lodestar configuration, run the command:

    yarn run cli beacon init \
    --eth1.enabled=false \
    --chain.genesisStateFile=genesis.ssz \
    --template-config-file witti.json \
    --root-dir .witti

    As a result, new files should had been created in .witti/beacon with a directory structure similar to:

    .witti/
└── beacon
    ├── beacon.config.json
    ├── chain-db/
    ├── enr.json
    └── peer-id.json

    Run a beacon node¶

    To start a Lodestar beacon run the command:

    yarn run cli beacon run \
    --root-dir .witti

    Immediatelly you should see confirmation that the different modules have started

    Info

    If your node is stuck with Current peerCount=0 review your network configuration to make sure your ports are open.

    A young testnet (such as Witti) should take a few hours to sync. If you see multiple or consistent errors in the logs, please open a Github issue or reach out to us in Discord. Just by reporting anomalities you are helping accelerate the progress of Eth2.0, thanks for contributing!
